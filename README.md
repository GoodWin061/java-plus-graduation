# Explore With Me

Приложение для публикации мероприятий и участия в них. Пользователи создают события, подают заявки, администраторы управляют категориями, подборками и модерируют события.

---

## Архитектура

Микросервисная архитектура с централизованной конфигурацией и discovery. Все сервисы регистрируются в Eureka, получают конфигурацию через Config Server. Внешние запросы идут через Gateway Server (порт `8080`).

**Инфраструктура:**
- **Discovery Server** (Eureka, порт `8761`) — реестр сервисов.
- **Config Server** — раздаёт конфигурации (native profile).
- **Gateway Server** — маршрутизация запросов к микросервисам.
- **User Service** - сервис для администратора - управления пользователями.
- **Event Service** - управляет событиями, категориями, подборками и локацией.
- **Request Service** - управляет заявками на участие в мероприятиях.
- **Comment Service** - управляет комментариями.
- **Stats Service** - собирает и анализирует статистику по запросам пользователей.

---

## Взаимодействие между сервисами

- **OpenFeign** - используется для надёжного и декларативного вызова REST-интерфейсов других сервисов — обеспечивает простой и типобезопасный способ межсервисного взаимодействия.
- **Eureka** - выступает в роли сервис-реестра: автоматически регистрирует доступные сервисы и позволяет им динамически находить друг друга, обеспечивая отказоустойчивость и балансировку нагрузки.
- **API Gateway** - является единой точкой входа для внешних клиентов: он маршрутизирует запросы к соответствующим микросервисам, скрывая внутреннюю структуру системы и обеспечивая централизованную обработку аутентификации, лимитов и логирования.

Каждый микросервис обладает изолированной базой данных на основе PostgreSQL, что гарантирует автономность и независимость развертывания. Взаимодействие между сервисами происходит исключительно через стандартизированные REST API, обеспечивая гибкость, масштабируемость и лёгкость поддержки системы.

## Фича
В качестве фичи были выбраны комментарии. Максиимальная длина которого составляет 1000 знаков. Создавать/редактировать/удалять комментарий может авторизованный пользователь, не являющийся автором мероприятия. Администратор модерирует комментарии перед публикацией.

### **Эндпоинты фичи**

| Метод  | Путь                                 | Описание                                  |
| ------ | :----------------------------------- |-------------------------------------------|
| POST   | /users/{userId}/comments             | Создать комментарий пользователя          |
| DELETE | /users/{userId}/comments/{commentId} | Удалить комментарий пользователя          |
| PATCH  | /users/{userId}/comments/{commentId} | Отредактировать комментарий пользователя  |
| DELETE | /admin/comments/{commentId}          | Удалить комментарий пользователя          |
| PATCH  | /admin/comments/{commentId}          | Отредактировать комментарий пользователя  |
| GET    | /admin/comments                      | Получить все комментарии                  |

## Конфигурация

Все конфигурации для микросервисов хранятся централизованно в **Config Server**. Взаимодействие настроено через Spring Cloud Config, который обеспечивает динамическое получение конфигурации.
